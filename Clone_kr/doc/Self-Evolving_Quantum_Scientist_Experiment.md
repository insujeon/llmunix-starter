# 자기 진화 양자 과학자 실험

## 요약

이 문서는 **LLMunix 순수 마크다운 운영체제 프레임워크** 내에서 에이전트 학습 패러다임의 "대통일"을 구현하는 획기적인 실험을 설명합니다. 이 실험은 고수준 아이디어에서 형식적 수학적 증명, 실행 가능한 양자 구현에 이르기까지 완전한 인간 과학적 발견 프로세스를 복제하는 에이전트 시스템을 만들고, **마크다운 기반 시스템이 경험에서 어떻게 학습**하여 시간이 지남에 따라 점점 더 지능적이 되는지를 보여줍니다.

**핵심 혁신**: 이 실험은 LLMunix의 기존 **지각적 상태 아키텍처** 및 **메모리 기반 학습** 기능을 활용하여 AI 시스템이 **런타임에 LLM이 해석하는 순수 마크다운 명세**를 통해 완전한 학습 생명주기를 구현할 수 있음을 보여줍니다.

## 개념적 기반

### 이중 프로세스 학습 모델

이 실험은 LLMunix의 기존 아키텍처 내에서 완전히 구현된 두 가지 보완적인 AI 학습 접근 방식을 합성합니다:

1. **단기 메모리 (실시간 작업 메모리)**:
   - LLMunix의 모듈형 상태 아키텍처 사용 (`workspace/state/` 디렉토리)
   - 특화된 파일: `plan.md`, `context.md`, `variables.json`, `history.md`, `constraints.md`
   - 실시간으로 특화된 서브 에이전트를 연결
   - 단일 실행 실행 내에서 동적 협업을 통해 복잡한 문제 해결
   - 모든 서브 에이전트 상호작용이 타임스탬프와 함께 `projects/[project]/memory/short_term/`에 기록됨

2. **장기 메모리 (통합된 지속적 학습)**:
   - LLMunix의 네이티브 `MemoryConsolidationAgent`가 완전한 에이전트 통신 추적 분석
   - `projects/[project]/memory/short_term/`에서 읽기 (에이전트 상호작용이 있는 세션 로그)
   - 패턴, 성공적인 워크플로우, 협업 전략 추출
   - 쿼리 가능한 지식으로 `projects/[project]/memory/long_term/`에 인사이트 통합
   - 향후 작업 실행을 개선하는 지속적 학습 시스템 생성
   - `QueryMemoryTool`을 통한 메모리 쿼리가 계획 및 구성요소 선택에 정보 제공

### Sutton-Karpathy 합성

이 아키텍처는 AI 개발의 근본적인 논쟁을 직접 다룹니다:
- **Rich Sutton의 입장** ("The Bitter Lesson"): 세계와의 지속적인 상호작용을 통해 직접적이고 검증 가능한 "실시간" 경험에서 학습 - 특별한 사전 훈련 단계 없이 즉석에서 학습하는 동물처럼
- **Andrej Karpathy의 입장** ("Summoning Ghosts"): 사전 훈련된 기초 모델을 초기화로 활용 (인간 데이터로 훈련된 LLM이 풍부한 매개변수 감독을 제공한 다음 경험적 학습으로 미세 조정)

자기 진화 양자 과학자는 두 접근 방식을 합성합니다: 기초 모델 기능을 초기화로 사용하면서 (Karpathy의 "유령") 마크다운 메모리 로그에 캡처된 경험적 학습을 통해 지속적으로 개선합니다 (Sutton의 "동물"로 이동). 이것은 "유령이 시간이 지남에 따라 더 동물 같아질 수 있음"을 보여줍니다 - LLM 기반이 지속적인 현장 학습을 지원할 수 있습니다.

## 시스템 아키텍처: 순수 마크다운 프레임워크

### LLMunix 운영 패러다임

**중요**: LLMunix는 **순수 마크다운 운영체제**로서:
- 모든 것이 마크다운으로 정의된 **에이전트** (의사결정자) 또는 **도구** (실행자)
- LLM 인터프리터가 런타임에 마크다운 명세를 읽고 해석
- **코드 생성 없음** - 시스템 동작이 LLM이 마크다운 문서를 해석하면서 나타남
- 모든 에이전트가 기능을 정의하는 YAML 프론트매터가 있는 마크다운 파일
- 도구는 Claude Code의 네이티브 기능에 매핑되는 마크다운 명세

## 핵심 원칙

이 실험은 여러 중요한 가설을 검증합니다:

2. **에이전트 특화가 마크다운을 통해 확장**
3. **메모리 아키텍처가 자기 개선 활성화**
4. **마크다운 기반 시스템이 자기 진화 가능**
