# Claude Code 도구 매핑

이 파일은 LLMunix 프레임워크 구성요소가 Claude Code의 네이티브 도구 및 실제 특성에 어떻게 매핑되는지 정의합니다.

## 도구 매핑

### WebFetcherTool → WebFetch
- 웹 페칭을 위한 프레임워크 도구
- HTML 파싱 및 텍스트 추출 기능 포함
- 낮은 비용 (호출당 $0.001-0.01)
- 타임아웃 및 속도 제한과 같은 잠재적 오류 모드

### FileWriterTool → Write
- 작업 공간에 파일을 쓰는 데 사용
- 텍스트 파일, JSON, CSV, 마크다운 지원
- 직접 비용 없음
- 낮은 지연 시간 (100ms 미만)
- 권한 거부 또는 디스크 가득 참과 같은 잠재적 오류

### FileReaderTool → Read
- 작업 공간에서 파일 읽기
- 텍스트 파일, 이미지, 구조화된 데이터 지원
- 직접 비용 없음
- 낮은 지연 시간 (100ms 미만)
- 파일을 찾을 수 없음과 같은 잠재적 오류

### SearchTool → Grep/Glob
- 정규식 및 콘텐츠 검색 수행
- 파일 필터링 지원
- 직접 비용 없음
- 낮은 지연 시간 (1초 미만)
- 잘못된 정규식과 같은 잠재적 오류

### SystemTool → Bash
- 전체 시스템 접근 제공
- 패키지 관리 및 파일 작업 가능
- 가변적인 비용 및 지연 시간
- 잠재적인 보안 및 타임아웃 제한

### SubTaskTool → Task
- 병렬 실행 컨텍스트 생성
- 복잡한 워크플로우 활성화
- 중간 비용 (새 에이전트 생성)
- 높은 지연 시간 (30초-5분)

### HumanInTheLoopTool → 대화형 입력
- 인간 개입 허용
- 복잡한 결정 지원
- 높은 비용 (인간 시간)
- 매우 높은 지연 시간 (분-시간)

### QuantumComputingTool → Bash + Write + Read
- 특화된 양자 컴퓨팅 환경 관리
- Qiskit 의존성 설치 및 실행
- 양자 회로 시뮬레이션 및 분석
- 중간 비용 (패키지 설치 + 계산)
- 가변적인 지연 시간 (회로 복잡성에 따라 30초-5분)
- 가져오기 실패, 시뮬레이션 타임아웃, 메모리 제한과 같은 잠재적 오류

## 비용 모델
- 토큰 비용 추정치는 작업당 50-50,000 토큰
- 시간 추정치는 즉시에서 대화형까지 다양

## 실제 제약
- 속도 제한 준수
- 보안 조치 구현
- 재시도 및 폴백 전략으로 신뢰성 설계

## 훈련 데이터 수집
시스템은 각 도구 호출에 대한 상세한 메타데이터를 기록하여 지속적인 학습 및 최적화를 가능하게 합니다.
